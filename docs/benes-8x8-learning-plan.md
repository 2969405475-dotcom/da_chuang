# 8x8 Benes 网络算法零基础学习计划

## 1. 这份计划解决什么问题

这份计划面向算法负责人，默认读者暂时不熟悉 Python、图论、光交换网络和半导体器件物理。

第一阶段只解决一个小而完整的问题：

```text
把 8x8 Benes 网络表示成一张有向无权图，
然后用 BFS 从一个输入端口找到一个输出端口的合法路径。
```

例如：

```text
输入：I3
输出：O6
输出：一条从 I3 到 O6 的合法节点列表及其跳数
```

第一阶段明确不考虑损耗、延迟、串扰、堵塞概率、真实波导长度和 KNN。这样做的目的不是回避实际问题，而是先把拓扑、路径和算法三个最基本概念分开学清楚。

## 2. 先记住四个核心概念

### 2.1 拓扑

拓扑只描述“谁能连接到谁”，不描述图上画出来的线有多长。

在本项目中，A 同学负责生成拓扑，算法部分只读取拓扑，不在 `routing.py` 中重新建图。

### 2.2 路径

路径是一个按连接方向排列的节点列表。例如：

```python
["I3", "S1_1", "S2_1", "S3_1", "S4_3", "S5_3", "O6"]
```

只有当列表中每两个相邻节点之间都存在一条有向边时，它才是合法路径。

### 2.3 最短路径

在无权图中，“最短”表示经过的边最少，也叫跳数最少：

```python
hop_count = len(path) - 1
```

它不表示波导的实际几何长度最短，也不表示损耗最低。

### 2.4 路由配置

“为 I3 到 O6 找一条路径”和“同时为 I0 到 I7 安排八组互不冲突的连接”是两个不同难度的问题。

- 单连接路径：第一阶段任务，BFS 可以完成。
- 完整输入输出置换：需要统一设置多个 2x2 开关，并处理资源冲突，是后续任务。

本地《Benes 网络的半自动选路法》主要研究第二种问题，不能直接等同于第一阶段的单路径 BFS。

## 3. 8x8 Benes 网络应掌握的结构

Benes 网络通常使用 `N = 2^n` 个输入和输出。对于 `N = 8`：

```text
n = log2(8) = 3
交换级数 = 2n - 1 = 5
每级交换单元数 = N / 2 = 4
交换单元总数 = 5 x 4 = 20
```

项目统一使用：

```text
输入端口：I0, I1, ..., I7
交换单元：S1_0, ..., S5_3
输出端口：O0, O1, ..., O7
```

结构可以先理解成：

```text
输入端口 -> 第1级 -> 第2级 -> 第3级 -> 第4级 -> 第5级 -> 输出端口
```

每个交换单元是一个 2x2 开关。后续研究开关状态时，通常区分：

- Bar：直通状态。
- Cross：交叉状态。

第一阶段只检查网络连接，不给 Bar 和 Cross 设置不同代价。

## 4. 相邻级之间的距离是否相同

这个问题必须分成两个模型回答。

### 4.1 第一阶段的图论模型

所有相邻级连接都视为一条无权边，每条边的代价都是 1。因此在算法眼中，相邻级之间的连接相同。

图上某条线画得更斜或更长，只是可视化布局，不代表它的边权更大。

### 4.2 后续的物理芯片模型

真实芯片中不一定相同。不同连接可能具有不同的：

- 波导实际长度；
- 弯曲数量和弯曲半径；
- 波导交叉次数；
- 传播损耗和交叉损耗；
- 串扰风险；
- 传播延迟。

因此后续可以把边从“无权边”升级为带属性的边：

```python
G.add_edge(u, v, length=..., crossing_count=..., loss=..., delay=...)
```

这时再使用 Dijkstra 计算总权重最低的路径。第一阶段不要编造这些物理参数。

## 5. 一个重要结论：BFS 在第一阶段能做什么

标准 8x8 Benes 是分级网络。一条完整的输入输出路径必须逐级向前，经过全部 5 个交换级。

按当前项目“输入端口和交换单元都是节点”的建模方式，一条完整路径通常包含：

```text
1 条输入到第1级的边
4 条交换级之间的边
1 条第5级到输出的边
合计 6 条边
```

所以多条合法输入输出路径很可能具有相同跳数。由此可得：

- BFS 可以验证输入和输出是否连通。
- BFS 可以返回一条最少跳数的合法路径。
- BFS 很适合学习队列、访问集合和路径还原。
- BFS 返回的第一条路径不一定在物理意义上最好。
- 当所有完整路径跳数相同时，BFS 不能区分它们的优劣。

这不是算法失败，而是因为“每条边权重都相同”的模型没有提供更多区分信息。

## 6. 实现 BFS 时要注意的参数和数据

### 6.1 输入参数

```text
G：A 同学生成的有向图
src：起点，例如 I3
dst：终点，例如 O6
```

### 6.2 BFS 内部数据

```text
queue：等待访问的节点队列
visited：已经访问过的节点集合
predecessor：记录每个节点从哪个节点到达
```

### 6.3 输出数据

```text
path：从 src 到 dst 的节点列表
hop_count：len(path) - 1
```

### 6.4 必须检查的异常

- `src` 不在图中；
- `dst` 不在图中；
- 起点不是输入端口；
- 终点不是输出端口；
- 拓扑错误导致不存在路径；
- 返回的路径中存在一对没有直接连边的相邻节点。

### 6.5 邻居顺序

如果一个节点有多个下一跳，BFS 会按照邻居被遍历的顺序返回其中一条并列最短路径。因此：

- 不要把某一条具体路径写死在测试里；
- 测试应检查路径合法、首尾正确、跳数最短；
- 为了让演示结果可重复，拓扑模块可以固定添加边的顺序，或算法中按节点名排序邻居。

## 7. 推荐的四周学习安排

默认每周投入 5 到 7 小时。每次学习控制在 40 到 90 分钟，并在当天留下一个可检查的小成果。

### 第 1 周：Python、图论和 Benes 结构

目标：能用自己的话解释节点、边、有向图、路径和 5 级 Benes。

第 1 次：学习 Python 列表、字典、集合和函数。

第 2 次：用字典手工表示一个六节点小图，并打印某个节点的邻居。

第 3 次：学习有向边和无向边的区别，理解光信号只能从输入级向输出级传播。

第 4 次：根据公式计算 4x4、8x8、16x16 Benes 的级数和交换单元数。

第 5 次：查看 A 同学的 8x8 拓扑图，逐级核对节点和连接方向。

本周验收：

```text
能解释为什么 8x8 有 5 级和 20 个交换单元；
能解释拓扑图线条的视觉长度不等于物理长度；
能区分单连接路径和完整置换路由。
```

### 第 2 周：先手写 BFS，再使用 NetworkX

目标：理解 BFS，不把 `nx.shortest_path()` 当成无法解释的黑盒。

第 1 次：学习先进先出队列，手工模拟从 A 节点开始逐层访问。

第 2 次：实现只返回访问顺序的 BFS。

第 3 次：增加 `predecessor`，从终点反向还原路径。

第 4 次：在小图上验证 BFS 得到的路径跳数最少。

第 5 次：在 Benes 图上调用统一接口：

```python
path = find_path(G, "I3", "O6", method="bfs", weight=None)
```

本周验收：

```text
path[0] == "I3"
path[-1] == "O6"
路径中每两个相邻节点之间都有有向边
```

### 第 3 周：验证结果，不急着增加新算法

目标：用实验回答“无权最短路径到底优化了什么”。

第 1 次：实现或调用路径合法性检查。

第 2 次：比较自己实现的 BFS 与 NetworkX 无权最短路径结果的跳数。

第 3 次：测试至少 8 组输入输出对，例如 `I0 -> O0`、`I0 -> O7`、`I3 -> O6`。

第 4 次：使用 `all_shortest_paths` 观察某一输入输出对是否存在多条并列路径。

第 5 次：记录每组请求的路径、跳数、成功与否和运行时间。

建议 CSV 字段：

```text
src,dst,method,path,hop_count,success,runtime_ms
```

本周验收：

```text
能说明为什么多条 Benes 路径可能具有相同跳数；
能说明 BFS 结果受邻居顺序影响，但最短跳数不受影响；
能用测试证明路径合法，而不是只看图片判断。
```

### 第 4 周：可视化、测试和阶段总结

目标：形成第一次可演示成果。

第 1 次：画出完整 8x8 拓扑，普通边使用灰色。

第 2 次：把 `I3 -> O6` 的路径边高亮为红色或蓝色。

第 3 次：在图标题或控制台中显示源端口、目标端口和跳数。

第 4 次：运行全部自动测试，修正拓扑或路径错误。

第 5 次：写阶段报告，明确当前模型的能力和限制。

本周验收成果：


## 8. 第一阶段测试清单

### 8.1 拓扑测试

- 8 个输入端口和 8 个输出端口均存在；
- 共 5 个交换级；
- 每级 4 个交换单元；
- 共 20 个交换单元；
- 边只从前一级指向后一级；
- 每个输入均能到达每个输出。

### 8.2 路径测试

- `find_path()` 返回 `list[str]`；
- 路径首节点等于 `src`；
- 路径末节点等于 `dst`；
- 路径中每一条边真实存在；
- BFS 跳数等于 NetworkX 给出的无权最短跳数；
- 输入不存在、输出不存在或无路可达时给出明确异常。

### 8.3 不应写死的内容

- 不写死 `I3 -> O6` 的节点序列；
- 不断言 BFS 必须返回某一条并列最短路径；
- 不把所有路径长度预先写成结论，先由测试验证拓扑模型；
- 不在 `routing.py` 中重新创建 Benes 拓扑。

## 9. 第一阶段结束后怎样升级

### 第二阶段：多连接和开关资源

研究多个输入输出请求同时存在时：

- 两条路径是否使用同一个交换单元端口；
- 开关的 Bar/Cross 状态是否冲突；
- 是否需要重排已有连接；
- 如何表示一组完整输入输出置换。

此时再认真学习 Waksman、循环选路、单边选路和中间级选路。

### 第三阶段：带权最短路径

在老师或器件同学提供可解释参数后，加入：

```text
waveguide_length
bend_count
crossing_count
switch_state
loss
delay
crosstalk
```

然后使用 Dijkstra，而不是继续使用无权 BFS 评价路径质量。

### 第四阶段：候选路径筛选与 KNN

先生成候选路由和真实或仿真的性能标签，再使用 KNN 判断路径属于“优”还是“劣”。

本地《基于贝纳斯光网络的 K 近邻路由优化算法研究》中的 KNN 是路径质量筛选方法，不负责替代 Benes 拓扑和基础路径生成。没有可靠训练标签时，不应把模拟标签包装成真实器件结论。

## 10. 建议阅读顺序

### 第一组：本阶段必须读

1. [NetworkX Shortest Paths 官方文档](https://networkx.org/documentation/networkx-3.6/reference/algorithms/shortest_paths/index.html)
   - 重点：无权最短路、BFS、`shortest_path`、`all_shortest_paths`。

2. [爱丁堡大学 Beneš Network 简介](https://homepages.inf.ed.ac.uk/rni/comp-arch/Networks/benes.html)
   - 重点：递归构造、2x2 开关、可重排的含义。

3. V. E. Beneš, [Optimal Rearrangeable Multistage Connecting Networks](https://onlinelibrary.wiley.com/doi/abs/10.1002/j.1538-7305.1964.tb04103.x), 1964.
   - 第一遍只读摘要和结构结论，不要求推导全部证明。

### 第二组：理解完整置换时再读

4. 陈国良、韩雅华，《Benes 网络的半自动选路法》，本地 PDF。
   - 重点：第 2 页的 8x8 结构图、Waksman 设置算法、单边选路思想。
   - 暂时跳过关联存储器的硬件细节。

5. Yao-Ming Yeh and Tse-yun Feng, [On a Class of Rearrangeable Networks](https://doi.org/10.1109/12.177307), 1992.
   - 重点：可重排网络与通用路由方案。

### 第三组：加入光学特性时再读

6. Jing Zhang, Huaxi Gu, Yintang Yang, [An Improved Distributed Routing Algorithm for Benes Based Optical NoC](https://arxiv.org/abs/1109.0752), 2011.
   - 重点：网络状态为什么会影响动态选路。

7. [Making Path Selection Bright: A Routing Algorithm for On-Chip Benes Networks](https://www.mdpi.com/2079-9292/13/5/981), 2024.
   - 重点：从单路径升级到完整路由表时的建模方式。

8. 赵莉等，《基于贝纳斯光网络的 K 近邻路由优化算法研究》，本地 PDF。
   - 重点：候选路径特征、训练标签和光学仿真验证。
   - 该文属于后期路线，不作为 BFS 入门材料。
